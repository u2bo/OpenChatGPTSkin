# Theme Studio（完整本地闭环）

Theme Studio 是 OpenChatGPTSkin 的本地三栏编辑器。它完成“创建草稿 → 隔离预览 → 校验 → 保存不可变版本 → 导入导出 → 应用到真实 Codex”的开发者闭环。

启动后首先进入主题首页：四个内置主题固定展示在“我的主题”，个人主题单独分区；选择主题后右侧展示名称、版本、作者、HTTPS 主题链接和描述。“激发创意”会加载所选主题并进入原有三栏编辑器。首页和编辑器共用中英文、浅色 / 深色偏好，并通过版本化本地偏好保存。

仓库地址默认是 `https://github.com/u2bo/OpenChatGPTSkin.git`。维护 fork 或镜像时可通过 `OPEN_CHATGPT_SKIN_REPOSITORY_URL` 覆盖；值必须是 `https://github.com/` 地址。

## 启动

在仓库根目录运行：

```powershell
npm install
npm run studio:dev
```

命令输出一行包含 `url` 的 JSON。打开该 URL 后，一次性 Fragment Token 会换取 `HttpOnly; SameSite=Strict` Cookie，并立即从地址栏和历史记录移除。Service 只监听随机 `127.0.0.1` 端口；按 `Ctrl+C` 停止。

如果要使用“应用到 Codex”，先通过 Codex 菜单或系统托盘完全退出普通实例。Runtime 只管理自己启动的 Codex，不会接管或强制结束已有实例。

Runtime 状态暂时不可读时，Theme Studio 仍允许本地编辑、恢复草稿和管理版本；只有“应用到 Codex”会返回明确的 Runtime 错误。

## 已实现

- 四套原创内置主题和个人主题版本；
- 左侧“主题库 / 编辑工具”双标签切换和中文配置名称；点击主题后会立即加载并在成功后自动切换到编辑工具；已有草稿时先选择加载或覆盖，取消则保持主题库不变；
- Theme Schema v2，以及旧 v1 文档的确定性读取迁移；
- 主色、面板色、边框和完整语义字体颜色；
- 背景图片、人物 / 前景图片、焦点、缩放、模糊、亮度和遮罩；
- 自动 / 浅色 / 深色外观、自动或手动文字安全区、任务页完整 / 氛围 / 横幅 / 关闭模式；
- 基础面板、菜单 / 弹层、终端的独立透明度和统一毛玻璃强度；
- 首页与任务工作区双视图预览，背景和 surface 视觉参数与 Runtime 使用同一生成模块；
- UI / 代码字体、字号、字重、缩放和 WOFF2 本地字体；
- 内置效果和用户图片装饰，最多 16 项；
- 侧栏密度、主视觉高度、卡片列数、输入框宽度、模块间距、顺序、尺寸、对齐和允许隐藏的模块；项目选择固定使用 Codex 官方几何，仅适配主题配色；
- 属性修改保留在当前编辑状态，不再定时写入草稿；刷新或重启 Studio 后自动打开最近草稿；
- 同一主题只保留一个草稿，重复打开时选择加载已有草稿或覆盖现有草稿；
- Schema、素材引用、4.5:1 文字对比度、受保护布局和 Runtime 编译大小校验；
- 只有点击“保存版本”才会写入个人主题版本；支持补丁版本递增，以及带即时反馈的单版本或整个个人主题删除；
- 带清单和 SHA-256 校验的 `.ocskin` 导入与导出；
- 精确 `{id, version}` Runtime 应用、热切换、启动应用和既有回滚语义；
- 真实 Runtime 状态与认证 SSE 更新。

完整模块布局会进入主题版本；允许调整的安全模块会在隔离预览和 Runtime 中生效。项目选择的显隐、尺寸、对齐和间距字段仅为格式兼容保留，运行时始终使用 Codex 官方大小、位置和层级。真实 Codex 会持续识别首页、任务路由、设置页、工作台面板、终端、应用菜单和弹层，并应用稳定的颜色、背景、透明度、毛玻璃、侧栏密度、输入框宽度、主视觉、建议卡和模块间距映射。

## 手动验收

1. 从四套内置主题分别创建草稿，切换“首页 / 任务工作区”预览，检查颜色、背景、字体、装饰和布局工具。
2. 选择一个内置主题，确认成功后自动进入编辑工具；如出现已有草稿提示，分别验证加载、覆盖和取消，再上传自己的背景图片。
3. 修改主文字、次要文字、链接、输入、占位和代码文字颜色；故意制造低对比度，确认保存与应用被阻止并定位到具体字段。
4. 上传 PNG/JPEG/WebP 背景、人物或装饰，以及 WOFF2 字体；确认预览立即更新。
5. 连续拖动同一滑杆或颜色选择器，确认不会自动保存；点击“保存版本”后再测试撤销、重做和刷新页面后重新打开草稿。
6. 保存版本，确认得到精确 `themeId@version`；继续编辑并保存，确认旧版本未被覆盖；测试删除单个版本和整个个人主题。
7. 导出 `.ocskin`，再导入并打开；确认主题与素材保持一致。
8. 完全退出普通 Codex，点击“应用到 Codex”；检查首页、真实任务、审阅 / 终端 / 浏览器 / 文件工作区、应用菜单和设置弹层均持续使用主题。
9. 使用 Runtime `pause`、`resume`、`switch` 和 `restore` 验证恢复路径。

## 安全边界

- UI 不直接访问文件系统、Named Pipe、CDP、Codex DOM、PID、调试端口或安装目录。
- 上传只接受用户通过浏览器文件选择器明确选择的二进制数据，不接受任意本地路径或网络 URL。
- 图片在本地旋转、缩放并转换为 WebP；字体只接受签名有效的 WOFF2。
- 草稿不能直接进入 Runtime；只有通过完整校验并安装到受控 `theme-store` 的不可变精确版本可以应用。
- Runtime 请求不能包含 CSS、HTML、JavaScript、DOM 选择器、路径或 URL。
- `sidebar`、`topbar`、`composer` 和 `content-layer` 始终保持可见。

## 当前不包含

- 可安装 Codex 插件、插件市场或主题市场；
- 安装器、Node.js SEA、托盘和开机启动；
- MCP、云同步、在线素材库或 AI 图片生成；
- 任意 CSS、HTML、JavaScript、DOM 选择器或自由坐标覆盖。
